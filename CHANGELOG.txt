Display Control+ Changelog
=========================

v1.0.11 (2026-03-23)
- Fixed duplicate tray icons by preventing dashboard startup registration from force-running the tray task
- Added a tray singleton lock so accidental second tray launches exit immediately
- Increased icon fill and sharpening for cleaner rendering on the Windows taskbar and tray

v1.0.10 (2026-03-22)
- Dashboard sets an explicit AppUserModelID and prefers packaged ICO for taskbar/process icon consistency
- Installer now removes legacy shortcut names so updates replace old links cleanly
- Installer triggers an icon cache refresh after install so new shortcut icons appear faster

v1.0.9 (2026-03-22)
- Desktop shortcut is now created automatically during install and uses the branded Display Control+ icon
- Improved ICO generation for sharper shortcut/taskbar icon rendering at common Windows icon sizes
- Dashboard/taskbar icon now uses the packaged Display Control+ .ico directly for consistent branding
- Detection mode selector labels simplified back to Input, Activity, and Both

v1.0.8 (2026-03-22)
- Rebuilt the app EXEs so the packaged app version matches the installer/release version
- Suppressed immediate repeat update prompts right after a self-update restart
- Made the build script fail on real non-zero exits instead of noisy tool output

v1.0.7 (2026-03-22)
- Installer now force-terminates DisplayControl.exe, tray.exe, and overlay_bg.exe before replacing files
- Fixes tray.exe file-lock errors during manual installs and updater-driven installs

v1.0.6 (2026-03-22)
- App icon now appears correctly on all EXEs, installer, taskbar, and desktop shortcut
- Auto-update now kills tray and overlay processes before launching installer, preventing "app already running" errors
- Installer EXE branding updated with Display Control+ logo

v1.0.5 (2026-03-22)
- Update now installs silently (no wizard shown) — app closes, updates, and reopens automatically
- Tray and dashboard relaunch after a silent update without user interaction
- App process gracefully closed before installer replaces files (CloseApplications)

v1.0.4 (2026-03-22)
- Updater dialog now stays above other windows (topmost + focused modal behavior)
- Added in-app update download and installer launch (no browser redirect)
- Added dashboard header version text under the Knight Logics brand/link
- Published as a new updater target so v1.0.3 clients can auto-detect and upgrade

v1.0.3 (2026-03-22)
- Added tray left-click behavior to open the dashboard directly
- Removed brief CMD flashes by launching scheduler/task/process commands with CREATE_NO_WINDOW
- Improved dashboard taskbar icon reliability using a generated .ico for native Windows taskbar display
- Implemented real Activity detection mode using per-monitor pixel-change sampling
- Clarified detection mode labels and help text for Input, Activity, and Both modes
- Published as a new version so in-app updater correctly prompts existing v1.0.2 installs

v1.0.2 (2026-03-22)
- Fixed frozen-launch recursion that could spawn repeated dashboard windows
- Hardened GUI single-instance locking for renamed packaged executables
- Fixed tray/runtime executable lookup for portable releases outside the source tree
- Added Windows multiprocessing freeze support to overlay_bg.exe startup
- Kept tray-first startup behavior as the recommended release workflow
- Fixed NameError crash in tray icon loader (BASE_DIR reference removed)
- Exit Completely now kills the full overlay process tree, preventing orphaned overlay windows

v1.0.1 (2026-03-19)
- Packaging cleanup release
- Replaced bloated dependency list with focused runtime/build requirements
- Standardized build output names to DisplayControl.exe and overlay_bg.exe
- Improved build script and task setup reliability
- Updated README with clear use case and feature-focused product positioning

v1.0.0 (2025-07-23)
- Initial professional release
- Multi-monitor overlay and screensaver support
- Modern tray UI with dark theme
- Robust installer with branding and auto-launch
- Logging and error handling
